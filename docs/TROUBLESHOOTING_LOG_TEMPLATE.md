# Troubleshooting Log Template

## 1. 目标

这份模板用于在测试和排障时，统一记录关键失败点。

作用：

- 让同类问题可复盘
- 避免“这次修好了，下次还得重新猜”
- 为后续自动化测试和错误码治理提供素材

---

## 2. 使用方式

每遇到一个关键失败点，新增一条记录。

建议按模块拆记录：

- pipeline
- standalone
- xai
- publish

---

## 3. 模板

```text
[问题标题]
标题生成失败但前端未明确提示

[日期]
2026-03-25

[模块]
standalone

[触发接口]
POST /api/generate-vertical-standalone

[失败阶段]
titling

[现象]
视频可以进入竖屏流程，但标题不像模型生成，表现为直接截取字幕前几句。

[预期]
标题必须由大模型生成；如果失败，接口应明确报错。

[实际]
标题生成脚本失败后走了字幕兜底，用户误以为标题是模型生成成功。

[关键日志]
generate_title.py failed: ...

[初步原因]
generate_title.py 调用路径错误，或模型调用失败后被兜底逻辑掩盖。

[影响范围]
standalone
xai vertical queue

[修复建议]
1. 禁用字幕兜底
2. 标题失败时直接返回 TITLE_GENERATION_FAILED
3. 前端展示明确错误提示

[验证方式]
1. 不填写标题执行 standalone
2. 模型成功时应返回真正标题
3. 模型失败时应明确报错

[结论]
待修复 / 已修复 / 已验证
```

---

## 4. 推荐记录字段

- 问题标题
- 日期
- 模块
- 接口
- 失败阶段
- 现象
- 预期
- 实际
- 关键日志
- 初步原因
- 影响范围
- 修复建议
- 验证方式
- 当前结论

---

## 5. 推荐按模块积累的失败点

### 5.1 Pipeline

- ComfyUI 无法连接
- workflow 字段失配
- build_video.py 失败
- 输出视频未生成

### 5.2 Standalone

- 上传成功但 ASR 失败
- 字幕转换失败
- 标题生成失败
- 竖屏渲染失败

### 5.3 XAI

- 账号池为空
- API key 缺失
- 外部接口限流
- 结果文件为空或未翻译

### 5.4 Publish

- 素材池为空
- 平台配置缺失
- 微信账号无效
- RPA 启动失败
- 浏览器 profile 异常

---

## 6. 建议落地方式

你可以后面任选一种：

- 手工维护一个排障日志 Markdown
- 每次测试失败后追加到同一个文档
- 以后再把这套模板沉淀成 issue 模板或测试记录模板
