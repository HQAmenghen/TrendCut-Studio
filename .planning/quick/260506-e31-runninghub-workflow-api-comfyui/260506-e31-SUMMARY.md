# Quick Task Summary: 接入 RunningHub workflow API 并保留原生 ComfyUI 模式

## Result

素材驱动数字人生成现在支持两种渲染方式：默认原生 ComfyUI HTTP API，以及 RunningHub 已发布 workflow API。

## What Changed

- 新增 RunningHub API client，支持资源上传、`/run/workflow/{workflowId}` 提交、任务查询轮询和输出视频 URL 提取。
- 新增 avatar renderer provider 分派层，原生 ComfyUI 继续走 `/upload/image`、`/prompt`、`/history`，RunningHub 走已发布 workflow 的 `nodeInfoList` 输入替换。
- 扩展素材驱动前端高级设置，允许选择渲染方式并配置 RunningHub API Base URL、workflow ID、输入节点和输出节点。
- 扩展任务状态字段，但不把临时 RunningHub API Key 持久化到 `task_state.json`。

- `npm run build:front` passed.
