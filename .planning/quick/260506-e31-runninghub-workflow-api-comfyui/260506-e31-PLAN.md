# Quick Task 260506-e31: 接入 RunningHub workflow API 并保留原生 ComfyUI 模式

## Goal

Add a selectable avatar rendering provider so operators can keep the existing native ComfyUI HTTP API path and also call a published RunningHub workflow through `/openapi/v2/run/workflow/{workflowId}`.

## Tasks

1. Add tested RunningHub API helpers for upload, workflow submission, polling/query parsing, and output URL extraction.
2. Extend material-driven avatar config/state and UI payloads with provider, RunningHub endpoint, workflow ID, instance type, and input node mapping while avoiding persisted API keys.
3. Route automatic avatar generation through the selected provider in both manual material-driven runs and autopilot starts, then verify with focused Jest tests and lint/build where feasible.
