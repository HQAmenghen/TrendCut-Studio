# Quick Task 260507-kr5: Auto-Pilot dual publish modes

## Goal

Support Auto-Pilot running both publishing pipelines at the same time:

- `vertical`: no digital human
- `avatar`: with digital human

Both modes must be allowed to create scheduled WeChat publish jobs, and due scheduled jobs should not get stranded when another job is already using the same account.

## Tasks

1. Extend publish config and frontend controls to persist/select multiple Auto-Pilot pipeline modes while preserving the existing single `pipelineMode` fallback.
2. Update the scheduler so each enabled mode has an independent active key, render queue metadata, duplicate check, and created publish job metadata.
3. Keep scheduled jobs in `scheduled_wait` until RPA successfully starts, so concurrent due jobs can retry on the next scheduler tick.
4. Add focused Jest coverage for dual-mode enqueue/job creation and scheduled account contention retry behavior.
